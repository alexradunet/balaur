## Why this matters

Balaur's `nodes` collection is "the unified knowledge spine" (`internal/nodes`),
but its `type` field is a **closed `SelectField` enum** —
`note/memory/skill/journal/person/book/idea/place`. Adding a new object type
(the defining feature of Capacities/Anytype: *"define your own object type"*)
today requires a code edit and a migration. Worse, the design dossier
(`plans/160-163-knowledge-graph-design-dossier.md` line 49) and the running
self-description (`internal/self/knowledge.md`) both **claim the type is
"extensible"** — it is not. This plan makes that claim true: `type` becomes an
open string validated against a `node_types` registry collection (a config
sibling to `llm_models`/`owner_settings`), seeded with the eight existing types.
After this, a new type is one registry row, and later plans (165 property
schemas, 167 tasks-as-nodes, 168 life-log-as-nodes) build on it.

This plan does **identity only** — a type's name, label, icon, and consent
default. Property schemas and templates are plan 165.

### Step 2: Create the `node_types` registry collection (migration up)

In the new migration file, package `migrations`, register `up`/`down`:

```go
func init() { m.Register(upNodeTypes, downNodeTypes) }
```

`upNodeTypes(app core.App) error` does two things:

**(a) Create the `node_types` collection** (owner-editable config). Fields:
- `name` TextField, Required, Max 60 — the type key, e.g. `"book"`. Unique index.
- `label` TextField, Max 120 — display name, e.g. `"Book"`.
- `icon` TextField, Max 20 — an emoji/glyph (optional).
- `born_status` SelectField, Required, MaxSelect 1, Values `["active","proposed"]`
  — the consent default for agent/model-created nodes of this type.
- `system` BoolField — true for built-in types the owner must not delete.
- `created` / `updated` AutodateField (match the baseline's pattern).

Use `setOwnerRules(col, types.Pointer(ruleOwner))` so the owner can manage types
via REST/dashboard. `ruleOwner` and `setOwnerRules` are exported from the
baseline file in the same package — reuse them, do not redefine. Add a unique
index `idx_node_types_name` on `name`.

**(b) Seed the eight current types.** Insert one `node_types` row per existing
type. `born_status` is `proposed` for `memory` and `skill` (consent-gated),
`active` for the rest. `system = true` for all eight. Suggested labels/icons:

| name | label | born_status | system |
|---|---|---|---|
| note | Note | active | true |
| memory | Memory | proposed | true |
| skill | Skill | proposed | true |
| journal | Journal | active | true |
| person | Person | active | true |
| book | Book | active | true |
| idea | Idea | active | true |
| place | Place | active | true |

(Icons optional; pick simple emoji or leave empty.)

**(c) `downNodeTypes`** deletes the `node_types` collection (mirror
`dropCollections` style: find by name, delete, ignore not-found).

### Step 3: Open `nodes.type` from enum → registry-validated string

Extend `upNodeTypes` (same migration, after the registry is created and seeded)
to convert the `nodes.type` field from `SelectField` to `TextField`,
**preserving existing data**.

PocketBase stores both `SelectField` and `TextField` as a SQLite `TEXT` column
named `type`, so the underlying column data is compatible. The safe, explicit
sequence:

1. `nodesCol, err := app.FindCollectionByNameOrId("nodes")` (return on error).
2. Read every existing distinct `type` value first, so you can assert no data
   loss later: `app.FindRecordsByFilter("nodes", "", "", 0, 0, nil)` and collect
   `r.GetString("type")` counts into a map. Log the counts via `app.Logger()`.
3. Replace the field definition by name:
   ```go
   nodesCol.Fields.RemoveByName("type")
   nodesCol.Fields.Add(&core.TextField{Name: "type", Required: true, Max: 60})
   ```
   Then `app.Save(nodesCol)`.
4. After save, re-read the records and assert the per-type counts are unchanged.
   If any count changed, **return an error** (`fmt.Errorf("nodes: type column
   data lost during conversion: ...")`) so the migration fails loudly rather
   than silently dropping the owner's data.

> If PocketBase's `RemoveByName`+`Add` drops the SQLite column data (counts
> change), STOP and report — do not proceed; the conversion needs a
> read-into-memory / write-back-after backfill instead, which is a design change
> worth surfacing.

The field stays `Required` (a node must have a type). Validation that the type
EXISTS in the registry is enforced in Go (Step 4), not by the DB — this is the
deliberate trade for extensibility.

**Verify**:
```
go test ./migrations/ -run TestSchemaBaseline -v 2>&1 | head -40
```
Build compiles; the migration runs without panic. (Assertion updates come in
Step 5.)

### Step 4: Add registry read + validation helpers in `internal/nodes`

Create `internal/nodes/types.go` (package `nodes`):

- `TypeNames(app core.App) ([]string, error)` — all registered type names
  (`app.FindRecordsByFilter("node_types", "", "name", 0, 0, nil)`), used to
  build the AI allow-list and validate.
- `TypeExists(app core.App, name string) (bool, error)` —
  `app.FindFirstRecordByFilter("node_types", "name = {:n}", dbx.Params{"n": name})`,
  treating not-found as `(false, nil)`.
- `BornStatus(app core.App, typ string) (string, error)` — returns the
  registry row's `born_status` for `typ` (default `StatusActive` if the row
  omits it). Later plans use this to decide a created node's initial status.
- `OwnerAuthoredTypes(app core.App) ([]string, error)` — type names whose
  `born_status = active` (the set the AI may create directly via `node_write`).
  This replaces the hardcoded `nodeTypes` slice.

Then make `nodes.Create` (in `nodes.go`, line 85) validate the type against the
registry: after the `title` check, call `TypeExists`; if the type is unknown,
return `fmt.Errorf("nodes: unknown type %q (not in node_types registry)", typ)`.
This keeps the spine honest now that the DB no longer enforces the enum.

> Keep `Create`'s signature unchanged. Only add the validation call.

**Verify**: `go test ./internal/nodes/ -v` → all pass (including the new
`types_test.go` from the Test plan).
