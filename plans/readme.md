## Findings considered and rejected (do not re-audit)

- **Recap midnight `||Equal` boundary** (`generate.go:197`): correct — a
  period whose `End` equals `now` is exactly complete; both loop and
  `ensureOne` guards agree.
- **Weekly multi-day streak lapse** (`streak.go`): matches the documented
  "one period of slack" semantic; by design.
- **`balaur.http` reaching localhost** (`vm.go:188`): explicit in-code
  design decision ("a personal box talks to its own services"); only the
  redirect behavior was undecided → folded into plan 010.
- **OS-tool path scoping** (`tools/os.go`): path limits on read/write/edit
  are theater while `bash` ships in the same opt-in set; OS access is the
  owner's documented full-OS grant.
- **"Any auth token can read api_key via REST"**: false as stated — heads
  are a separate auth collection; `ruleOwner` restricts REST to the owner's
  `users` token. The defense-in-depth `Hidden:true` lives in plan 010.
- **"fts5 spike test broken at ncruces v0.34.4"**: false — the baseline run
  at c4fce47 passes `internal/search`.
- **x/crypto, x/net lag**: already at their security-fix versions
  (v0.52.0 / v0.55.0); informational.
- **Unbounded chat history page load**: false — history is windowed to 60
  (`web.go:154`).
- **Per-request template parsing**: false — templates parse once at
  `Register` (web.go:79).
- **`users` single-record guard**: rejected — user creation already
